# BUNKER — Boot Sequence (Discussion Notes)

Goal: align on tone, duration, emotional objective, sound doctrine, and the exact moment control transfers to the operator.
This document proposes a default direction to approve or modify.

---

## 1) Proposed Default Direction (Recommended)

### Tone
**Institutional procedural calm** with a subtle undercurrent of consequence.
Not mythic. Not playful. Not comedic.

### Emotional Goal
- “I am entering a hardened facility.”
- “This system is old, powerful, and reliable.”
- “I am in control — and accountable.”

### Duration
Two-tier boot (so it stays usable daily):
- **STANDARD BOOT:** 10–14 seconds (daily)
- **CEREMONIAL BOOT:** 35–55 seconds (optional toggle)

### Sound vs Silence
- Start with near-silence + faint facility hum
- Mechanical confirmation sounds: relays, latches, lamp clicks
- One restrained chime at handoff (not celebratory)

### Control Handoff
Control is granted only after:
1) security check pass
2) storage/index integrity checks complete
3) routing table loaded
4) observability online
Then: **OPERATOR CONTROL ENABLED**

---

## 2) Boot Sequence Structure (Macro Beats)

### Beat A — Wake
- Black screen → faint scanline emergence
- Global state lamp appears: **OFFLINE → PROCESSING**

### Beat B — Containment & Access
- ACCESS GATE: checking tokens / local operator key
- If unauthorized: “ACCESS DENIED” (hard cut; no comfort copy)

### Beat C — Core Systems Spin-Up
Initialize subsystems as physical bays:
- POWER BUS
- CORE BAY
- ARCHIVE STACKS
- VECTOR VAULT
- ROUTER CONSOLE
- OBSERVATION DECK

### Beat D — Integrity & Diagnostics
- quick checksum-like steps with timestamps
- failures generate an **INCIDENT** card with remediation

### Beat E — Command Theater Online
- Location path locks in: **OPS / COMMAND THEATER / MAIN CONSOLE**
- State lamp settles to **STABLE**

### Beat F — Handoff
- “OPERATOR CONTROL ENABLED”
- UI remains still for 300–500ms (weight)
- Cursor/interaction becomes available

## 4) Subsystem Mapping (Diegetic → Implementation)
*(Names are diegetic; this is for internal alignment.)*

### ACCESS GATE
- Auth + org gating

### SECRETS CAGE
- Secrets store; keys; tokens

### ROUTER CONSOLE
- Routing policies; fallbacks; model selection

### CORE BAY
- Local serving + tools runtime

### VECTOR VAULT
- Embeddings store

### ARCHIVE STACKS
- Relational records + snapshots

### OBJECT LOCKER
- Blob/object storage

### OBSERVATION DECK
- Traces, run history, cost/readouts

### ALERT LADDER
- Incident escalation

## 5) Visual Language (Boot Only)
- CRT emergence (subtle) + instrument framing
- Large numeral/zone indicator appears once (ceremonial)
- Stamps: “VERIFIED”, “MOUNTED”, “SEALED” (used sparingly)

## 6) Variants (If you want a different flavor)

### Variant 1 — Grounded Industrial (Default)
- fastest, most usable daily

### Variant 2 — Corporate Propaganda Layer
- adds a short “safety doctrine” line during boot
- risk: feels too performative if overdone

### Variant 3 — Silent Ritual
- no narration; only hum + relays
- strongest “authority” option
